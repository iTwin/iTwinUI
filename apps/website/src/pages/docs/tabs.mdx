### Composition API

You can build Tabs with subcomponents that are fully customizable.

We provide:

- `Tabs.Wrapper`
- `Tabs.List`
- `Tabs.Tab`
- `Tabs.TabIcon`
- `Tabs.TabLabel`
- `Tabs.TabDescription`
- `Tabs.Actions`
- `Tabs.Panel`

`Tabs.Tab` and `Tabs.Panel` can be associated with each other by passing them the same `value`.

### Props API

You can use Tabs directly with labels, actions, and children as panels. Although this pattern can suffice for simple cases, we strongly recommend using composition API for the best experience.

A list of all available props can be found at the end of this [page](#props).

### Borderless

As their name implies, borderless tabs have no frame or stroke surrounding them. The label appears on top of the content, above a stroke that separates it from the text area. The stroke cuts off beneath the active tab, so it can be visually associated with its respective content.

</LiveExample>

### Pill

Pill tabs are meant to be used with content that belong to a similar category. They act like a filter for toggling through different states of similar content.

## Usage

- Use [borderless tabs](#borderless) when the tabbed content can be easily told apart from its surroundings (or lack thereof).
- The maximum recommended amount of [pill tabs](#pill) is 4.
