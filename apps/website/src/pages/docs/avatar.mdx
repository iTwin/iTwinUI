The avatar is an essential UI component, making it easier to identify a person or organization by their photo, icon, or initials. A user’s photo helps put a face to the name and makes it more personal. Any time an app requires a user profile, an avatar should be included.

## Appearance

### Sizes

There are four sizes of avatars:

<LiveExample src={frontmatter.exampleCodeSizes} />

- `small`
- `medium`
- `large`
- `x-large`

Which size to use is at your discretion. Determine which size is best for your context based on available space.

### Initials

If a user does not upload a photo, the user’s first & last initials are displayed. You can display a maximum of 2 letters and a minimum of 1 letter. In the off chance there are user accounts without any name associated, you can display the first letter of the user’s email address.

<LiveExample src={frontmatter.exampleCodeInitials} />

### Icon

An icon may be display instead of initials, typically used by organizations instead of a person.

<LiveExample src={frontmatter.exampleCodeIcon} />

### Picture

If a user uploads a photo we display their photo instead of their initials.

<LiveExample src={frontmatter.exampleCodePicture} />

### Status indicators

You may want to indicate the online status of a user. Status indicators display in the bottom right of the avatar. We suggest not using statuses on the small size avatar. The available statuses are:

- `online`
- `away`
- `busy`
- `offline`

<LiveExample src={frontmatter.exampleCodeStatuses} />

## Avatar group

Avatars can be displayed in a list to represent a group of people. This visual treatment reinforces the fact that these people work together and/or share similar permissions to access/view/edit a file or a project.

<LiveExample src={frontmatter.exampleCodeGroup} />

### Stacked

To save on space, the avatars can overlay each other. Note that when using the stacked group that you can not display [status indicators](#status-indicators).

<LiveExample src={frontmatter.exampleCodeGroupStacked} />

### Animated stack

You can also have the stacked group animate to unstacked on hover. When doing this, be sure to pay close attention to unintentionally shifting other content on the page.

<LiveExample src={frontmatter.exampleCodeGroupAnimated} />

### Overflow

A maximum of six avatars can be shown within a group. Any additional avatars will not be shown and instead the last avatar will display a count of how many avatars are not shown. To display all of the hidden users, we recommend the user interact with the numbered avatar. You could display a [tooltip](tooltip) on hover, a [dialog](dialog) on click, or take the user to a new page on click.

<LiveExample src={frontmatter.exampleCodeGroupOverflow} />
